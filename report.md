# 📄 Pretext 기반 오픈소스 확장 프로젝트 보고서

## 1. 프로젝트 제목
**Pretext 3D Globe: 실시간 지구 데이터 시각화를 위한 3차원 공간 투영 및 동적 텍스트 레이아웃 엔진**

---

## 2. 프로젝트 개요 및 개발 목적
본 프로젝트는 기존 2차원 평면 상의 텍스트 레이아웃 한계를 극복하고, 대량의 실시간 지구 데이터를 웹 환경에서 부드럽게 시각화하기 위해 개발되었습니다. 
* **해결하고자 하는 문제:** 3D 공간 상에서 수천 개의 개체(위성, 도시 등)를 표현하고 실시간 텍스트 라벨을 렌더링할 때, 기존 브라우저의 DOM API(`getBoundingClientRect`, `offsetHeight` 등)를 사용하면 프레임마다 레이아웃 리플로우(Layout Reflow)가 발생하여 심각한 성능 저하가 일어납니다.
* **해결 방안:** 웹 하드웨어 가속 3D 렌더링 엔진인 **Three.js**와 성능 중심의 DOM-free 텍스트 레이아웃 엔진인 **Pretext**를 결합하였습니다. 3D 공간 상의 좌표를 실시간으로 2D 화면 좌표로 투영하고, Pretext를 이용해 오프스크린에서 텍스트의 크기와 줄바꿈을 계산한 뒤, 단 하나의 2D Canvas 오버레이 레이어에 60fps로 일괄 드로우하는 고성능 시각화 대시보드를 구축했습니다.

---

## 3. 기존 Pretext 코드에서 활용 및 확장한 부분
본 프로젝트는 Pretext 라이브러리의 제공 데모 중 **[Editorial Engine (에디토리얼 엔진)]**을 단일 타겟으로 선택하여 확장했습니다.

* **기존 Editorial Engine 데모의 핵심 컨셉:**
  * 2차원 평면 Canvas 위에 떠다니는 원형의 장애물(Obstacle) 좌표를 인식하여, 텍스트가 장애물 영역을 침범하지 않고 그 주변으로 흘러내리도록 실시간으로 라인을 분할하고 줄바꿈하는 기술입니다.
* **본 프로젝트에서의 확장 (3차원 공간 및 동적 장애물 회피 시스템):**
  * **장애물 영역의 3차원 공간 확장:** 2D 평면에 머물러 있던 원형 장애물의 개념을 3차원 공간으로 대폭 확장했습니다. 회전하는 지구 구체(지표면)와 우주 공간을 공전하는 실시간 위성 데이터의 3D 월드 좌표를 브라우저 카메라 행렬을 통해 2차원 화면 좌표로 실시간 투영(`projectToScreen`)하여 동적인 원형 장애물 영역(Obstacle Area)으로 변환합니다.
  * **동적 텍스트 충돌 회피 (Dynamic Obstacle Avoidance):** 투영된 수백 개의 데이터 노드를 dynamic obstacle로 규정합니다. `TextLayer.ts`에서 Pretext의 `prepareWithSegments`와 `layoutWithLines`를 통해 예측된 텍스트 박스 크기를 바탕으로 실시간 Bounding Box를 계산(AABB 충돌 감지 알고리즘)합니다. 줌 레벨과 우선순위(ID 기반)에 따라 글자들이 서로 겹치거나 지구 뒤편으로 넘어간 개체는 화면에서 제외하고, 장애물과 글자가 겹치지 않는 최적의 레이아웃을 계산하여 60fps로 동적 배치합니다.

## 4. 구현한 주요 기능 설명 (3개 이상)
1. **🛰️ 실시간 위성 궤도 수학적 시각화 및 추적 (Stuff in Space 오마주)**
   * Celestrak 실시간 TLE(Two-Line Element) 궤도 데이터를 활용하여, 우주 공간 상에 떠 있는 인공위성들의 위치를 수학적으로 시각화했습니다. 
   * 위성을 마우스로 클릭하면 `satellite.js` 물리 엔진을 통해 해당 위성의 실시간 미래 예상 궤도 경로(Orbit Path Line)를 3차원 상에 녹색의 루프로 그려냅니다.
2. **🏙️ 지표면 인구밀도 기반 3D 스파이크(Spike) 데이터 시각화**
   * 정적 GeoJSON 데이터로부터 전 세계 주요 도시의 좌표와 인구 밀도 데이터를 파싱하여 지표면 위에 3D 막대(Spike) 형태로 렌더링했습니다.
   * 인구 밀도가 높을수록 더 높은 기둥과 함께, 텍스트 레이블의 크기와 배치 빈도를 Pretext를 통해 계산하여 정보 밀도를 다르게 분기시켰습니다.
3. **🗺️ 3D-2D 하이브리드 가변 맵 모핑 (Unroll to 2D Map)**
   * 구체 형태의 3D 지구를 평면 2D 지도로 부드럽게 펼쳐주는 모핑 애니메이션을 물리 엔진 연산(`Math.atan2` 프로젝션)으로 구현했습니다.
   * 모핑 과정에서 지구의 구체 메쉬 및 대기권(Atmosphere glow) 메쉬의 투명도(Opacity)가 부드럽게 0으로 수렴하여 사라지고, 오직 데이터 라벨과 경계선 네트워크 아크만 평면 공간에 압착되어 HUD 스타일의 디지털 지도로 완벽하게 전환됩니다.

## 5. 본인이 추가한 기능 중 가장 가치 있는 기능 (대표 기능)
**"사용자 정의 실시간 상세 정보 패널 (위성 궤도 추적 및 도시 대시보드)"**
* **사용자 가치:** 단순히 점들이 움직이는 관찰용 지도를 넘어, 사용자가 특정 대상과 상호작용하여 정보를 획득하는 실용성을 부여했습니다.
* **동작 원리:** 사용자가 지표면의 도시(예: Seoul)나 우주의 위성을 클릭하면, Three.js Raycaster와 TextLayer의 2D HitTest가 동시 작동하여 객체를 타겟팅합니다.
* **제공 정보:** 
  * **위성 클릭 시:** 실시간 고도(Altitude), 정확한 현재 위/경도 좌표 및 궤도선 표시.
  * **도시 클릭 시:** 해당 도시의 인구 수 데이터뿐만 아니라, **실시간 날씨 정보 API** 및 **실시간 뉴스 헤드라인 API**를 연동하여 해당 도시에 대한 다채롭고 가치 있는 최신 정보 대시보드를 우측 홀로그램 패널 형태로 사용자에게 직관적으로 제공합니다.

## 6. UI/UX 개선 내용
* **글래스모피즘(Glassmorphism) 기반 HUD UI:** 사이드 패널 및 정보 패널을 어두운 반투명 아크릴 느낌의 디자인(Blur 필터 및 네온 시안 테두리)으로 제작하여 SF 우주선 통제실에 있는 듯한 심미성을 제공합니다.
* **마그네틱 스나이핑 (Magnetic Snapping) UI:** 사용자가 특정 위치 근처로 카메라를 돌리면, 중앙 크로스헤어 타겟으로 카메라가 자석처럼 부드럽게 스냅(Snap)되어 원하는 정보에 빠르고 정확하게 접근하도록 돕습니다.
* **텍스트 가독성을 위한 아웃라인 드로잉:** 검은 배경이나 지구 텍스처 위에서도 텍스트가 묻히지 않도록 Canvas 2D 상에서 글자에 두꺼운 검정 아웃라인(`strokeText`)을 입힌 후 채우기(`fillText`)를 수행하여 어떤 각도에서도 텍스트 가독성을 극대화했습니다.

## 7. 코드 구조 및 주요 구현 방식
* **Vite + TS + Three.js + Canvas 2D 하이브리드 렌더링 구조:**
  * `src/components/Globe.ts`: WebGL 및 Three.js 씬 빌드, 지구 메쉬 및 입자 시스템 관리, Raycasting 클릭 이벤트.
  * `src/components/TextLayer.ts`: 2D Canvas 위에 **Pretext 라이브러리**를 이용하여 텍스트 객체를 가상화 및 일괄 드로우하는 핵심 렌더링 루프 담당.
  * `src/workers/dataWorker.ts`: CPU 연산이 많이 필요한 TLE 궤도 계산, GeoJSON 인구 데이터 파싱 등을 메인 스레드와 완전히 분리하여 Web Worker 내부에서 병렬 처리함으로써 UI 버벅임(Jank)을 완벽 차단.

## 8. AI 도구 활용 내역
* **사용 도구:** Claude 3.5 Sonnet / Gemini 3.5 Flash
* **주요 프롬프트 예시:**
  * *"Pretext 라이브러리의 `prepare` 및 `layout` API를 활용하여 Three.js의 3D Vector3 좌표를 2D 화면 좌표로 변환하고 겹치지 않게 Canvas에 렌더링하는 클래스를 작성해줘."*
  * *"Stuff in Space처럼 인공위성을 클릭했을 때 satellite.js를 사용해 실시간 미래 궤도 포인트를 생성하고 Three.js Line으로 부드럽게 그리는 방법을 제안해줘."*
* **인간 개발자의 검토 및 수정 과정:**
  * **CORS 이슈 해결:** AI가 제안한 외부 CelesTrak API 실시간 페칭 코드가 브라우저의 CORS 제한으로 차단되는 문제를 감지하고, 실제 활성 위성 TLE 데이터를 `public/active.txt`에 정적 캐싱하여 빌드하는 방식으로 수동 우회 설계하였습니다.
  * **2D Unroll 성능 개선:** 모핑 중에 텍스트 레이블의 월드 좌표가 어긋나는 것을 방지하기 위해 `TextLayer.ts` 내부의 보간 공식(`new THREE.Vector3().lerpVectors`)을 추가 작성하여 모핑 애니메이션 도중에도 텍스트가 찢어지지 않고 정확하게 추종하도록 보정했습니다.

## 10. 한계점 및 향후 개선 방향
* **인방향 연결성 의존:** 실시간 날씨 및 뉴스 API는 외부의 무료 엔드포인트를 사용하므로 오프라인 환경이나 API 제공처의 트래픽 제한에 따라 데이터 갱신이 불가능할 수 있어, 로컬 모크 데이터 세트로의 폴백(Fallback) 구조가 상용화 시 추가 보완되어야 합니다.
* **위성 렌더링 최적화:** 현재 수천 개의 위성을 Three.js Points로 표현하나, 개별 위성의 정밀한 물리 궤도를 전수 실시간 계산 시 CPU 연산 오버헤드가 증가할 수 있어 Web Worker 내부의 연산 주기를 더 최적화하거나 GPU 셰이더(GPGPU)로 이관하는 방안을 고려 중입니다.
